# Скульптор Лица - Backend

## Стек

- **Runtime:** Node.js 20 LTS
- **Язык:** JavaScript (CommonJS, `require`)
- **Фреймворк:** Express 4
- **БД:** PostgreSQL 15+ (драйвер `pg`, пул соединений)
- **Аутентификация:** JWT (jsonwebtoken, HS256), bcryptjs (12 раундов)
- **Email:** Nodemailer (SMTP)
- **Загрузка файлов:** Multer
- **Безопасность:** Helmet, CORS, express-rate-limit
- **Dev:** Nodemon
- **Деплой:** PM2 + Nginx reverse proxy на Ubuntu VPS

## Структура проекта

```
backend/
├── src/
│   ├── index.js          — Точка входа, запуск сервера
│   ├── app.js            — Express app, middleware, маршруты
│   ├── db/index.js       — PostgreSQL pool
│   ├── db/migrations/    — SQL миграции (запускаются вручную)
│   ├── middleware/        — Кастомные middleware (auth)
│   ├── routes/            — Роуты (auth, user)
│   ├── services/          — Бизнес-логика (email, token)
│   └── utils/             — Утилиты (jwt)
├── public/avatars/        — Хранение аватаров пользователей
└── .env.example           — Шаблон переменных окружения
```

## Практики и правила

### Код

- Весь код пишется на **JavaScript (CommonJS)**. Не использовать ESM (`import/export`), не использовать TypeScript.
- Один файл = один модуль. Экспорт через `module.exports`.
- Роуты группируются по доменной области в `src/routes/`. Каждый файл роутов экспортирует `express.Router()`.
- Бизнес-логика выносится в `src/services/`, не в роуты. Роуты занимаются только валидацией входных данных, вызовом сервисов и формированием ответа.
- Middleware кладётся в `src/middleware/`.

### База данных

- Прямые SQL-запросы через `pg` pool (`db.query()`). ORM не используется.
- Параметризованные запросы обязательны (`$1, $2, ...`). Никогда не конкатенировать пользовательский ввод в SQL.
- Миграции — чистые `.sql` файлы в `src/db/migrations/`, нумеруются последовательно (`001_`, `002_`, ...).
- Все таблицы имеют поля `created_at` и `updated_at` с дефолтным `NOW()`.

### API

- Все маршруты под префиксом `/api/`.
- Формат ответов — JSON. Успех: `{ success: true, ... }` или данные напрямую. Ошибка: `{ error: "сообщение" }`.
- HTTP-статусы: 200 (ОК), 201 (создано), 400 (невалидные данные), 401 (не авторизован), 404 (не найдено), 409 (конфликт), 500 (ошибка сервера).
- Сообщения об ошибках для пользователя — на русском языке.
- Email всегда нормализуется: `email.toLowerCase().trim()`.

### Аутентификация

- JWT передаётся в заголовке `Authorization: Bearer <token>`.
- Payload JWT: `{ userId, email }`.
- Защищённые маршруты используют middleware `requireAuth` из `src/middleware/auth.js`.
- Пароли хешируются через bcrypt с 12 раундами.
- Токены (magic link, reset) — одноразовые, помечаются `used_at` после использования.

### Безопасность

- Rate limiting на все auth-эндпоинты (10 запросов / 10 минут).
- Helmet включён для security headers.
- CORS ограничен `CORS_ORIGIN` из `.env`.
- Не раскрывать существование email при сбросе пароля (всегда отвечать успехом).
- Не коммитить `.env` файлы. Секреты только через переменные окружения.

### Обработка ошибок

- Каждый роут оборачивается в `try/catch`.
- Ошибки логируются через `console.error` с указанием эндпоинта.
- Пользователю возвращается общее сообщение, не stack trace.
- Глобальный error handler в `app.js` ловит необработанные ошибки.

### Валидация

- Входные данные валидируются в начале каждого роута.
- Минимальные проверки: наличие поля, формат email (`includes('@')`), длина пароля (>= 6 символов).
- При невалидных данных — ранний `return res.status(400).json({ error: ... })`.

### Файлы и загрузки

- Аватары сохраняются в `public/avatars/`, раздаются как статика через Express.
- Multer используется для обработки multipart/form-data.

### Переменные окружения

- Все конфигурации через `process.env`, загружаются из `.env` через `dotenv`.
- Обязательные переменные перечислены в `.env.example`.
- Порт по умолчанию: `4000`.

### Конвенции именования

- Файлы и папки: `camelCase` или одно слово строчными (`auth.js`, `email.js`).
- Таблицы БД: `snake_case`, множественное число (`users`, `auth_tokens`).
- Поля БД: `snake_case` (`password_hash`, `created_at`).
- Переменные JS: `camelCase`.
- Env-переменные: `UPPER_SNAKE_CASE`.

### Фронтенд-интеграция

- Фронтенд на Next.js ожидает API на `NEXT_PUBLIC_API_URL` (по умолчанию `http://localhost:4000/api`).
- CORS настроен на `http://localhost:3000` в dev-режиме.
- JWT возвращается в теле ответа (не в cookie). Фронтенд хранит его в localStorage.
