Live broadcasting

Функция позволяет администратору сайта вести прямые видеотрансляции для зрителей. Зрители открывают страницу, смотрят стрим и общаются в чате в реальном времени. Никакой регистрации для просмотра не требуется. Пользователь указывает только имя которое будет использоваться в чате

Стек

Бэкенд: Node.js + TypeScript + Express

node-media-server — принимает RTMP поток от OBS на порту 1935

FFmpeg — транскодирует RTMP в HLS сегменты на диске

socket.io — WebSocket для чата и статуса трансляции

.env — хранит STREAM_KEY для аутентификации администратора

Фронтенд: React + TypeScript

hls.js — воспроизводит HLS поток в браузере

socket.io-client — подключается к чату и получает статус стрима

План реализации

Шаг 1 — RTMP сервер + HLS транскодинг Поднять node-media-server, валидировать STREAM_KEY, запускать FFmpeg при подключении OBS. FFmpeg нарезает поток на .m3u8 + .ts сегменты и кладёт их на диск.

Шаг 2 — Express: раздача HLS и статус API Добавить роут /hls/* который отдаёт файлы из папки ./media как статику. Добавить GET /api/status который возвращает { online: boolean, startedAt: string | null }.

Шаг 3 — WebSocket Подключить socket.io к Express серверу. Emit stream:status при старте и конце трансляции. Принимать chat:message от клиентов и рассылать всем остальным.

Шаг 4 — React: VideoPlayer Компонент на hls.js который подключается к .m3u8 когда стрим онлайн и показывает заглушку когда офлайн. Статус получает через useStream хук.

Шаг 5 — React: LiveChat Компонент на socket.io-client. Показывает сообщения в реальном времени, отправляет новые. Отключается когда стрим офлайн.

Шаг 6 — Сквозной тест OBS начинает трансляцию → видео появляется в браузере через ~10 секунд. Два браузера видят сообщения чата друг друга. OBS останавливает трансляцию → плеер переходит в офлайн состояние.
